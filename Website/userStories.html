RWP User Stories


Guest Student Faculty Administrator Tester

RWP Guest

  • High Priority
    • I must be able to read the full material of a public artifact.

    • I must be able to login with valid credentials.

    • I must be able to filter and sort artifacts that belong to the database.

    • I must not be able to access private artifacts.

    • I must not be able to modify or update public artifacts.

    • I must not be able to edit or update artifacts.

    • I must not be able to access user accounts.

  • Low Priority
    • I must be able to obtain a student or faculty login and password.

    • I must be able to be invited by administrators or other users.

    • I must not be able to change a user’s access level.

    • I must not be able to invite other users.

    • I wish to be able to request access to private material.

    • I wish to be able to be notified of what type artifacts I can access (E.g. public/private).

RWP Student

  • High Priority
    • I must be able to do everything a Guest can do.

    • I must be able to login with valid credentials.

    • I must be able to filter and sort full contents of a file.

    • I must be able to access the most up to date information.

    • I wish to be able to tag artifacts with keywords describing their content.

    • I must be able to search for content by keywords.

    • I must not be able to change a user’s access level.

    • I must not be able to delete user accounts.

    • I must not be able to access private artifacts without invite.

    • I must not be able to edit or update artifacts I do not own.

  • Low Priority
    • I must be able to request access to private resources.

    • I must be able to access the database with my credentials off campus.

    • I wish to be able to bookmark artifacts and repositories I will use again.

    • I wish to be able to flag artifacts for review by the owner or administrator if they appear to conflict with other artifacts covering the same content.

    • I wish to be able to flag artifacts for review by the owner or administrator if they appear out-of-date.

    • I wish to be able to Invite new users.

    • I wish to be able to create and upload resources.

RWP Faculty

  • High Prioroty
    • I must be able to do everything a Student can do.

    • I must be able to login with my faculty credentials.

    • I must be able to upload artifacts.

    • I must be able to edit and update artifacts I own.

    • I must be able to set access level requirement for artifacts I own.

    • I must be able to create notifications for myself regarding regular updates to artifacts.

    • I must be able to request access to private resources.

    • I must not be able to access, edit, or update artifacts I don’t own.

    • I must not be able to delete user accounts.

    • I must not be able to change a user’s access level.

  • Low Priority
    • I must be able to access database off campus.

    • I wish to be able to bookmark artifacts I will use again.

    • I wish to be able to invite new users.

    • I wish to be able to access the source of any artifact.

    • I wish to be able to see usage reports about artifacts I own.

    • I wish to be able to tag artifacts with keywords describing their content.

    • I wish to be able to view information about the owner of an artifact.

RWP Administrator

  • High Priority
    • I must be able to do everything Faculty can do.

    • I must be able to Test administration.

    • I must be able to change attributes of any artifact, such as access level requirement.

    • I must be able to remove artifacts.

    • I must be able to create and change user accounts.

    • I must be able to view reports about users and artifacts.

  • Low Priority
    • I wish to be able to reactivate user accounts.

    • I wish to be able to recover deleted files .

    • I must not be able to delete user accounts without documentation.

    • I must not be able to reset passwords.

RWP Tester

  • High Priority
    • I must be able to access artifacts within my testing parameters.

    • I must be able to view reports about users and artifacts.

    • I must be able to have administrative capabilities within my testing parameters.

    • I must be able to analyze results.

    • I must be able to report test problems and anomalies.

    • I wish to be able to improve software quality.

    • I wish to be able to capture user requirements.

    • I must not be able to test my own products.

    • I must not be able to make all the decisions and changes to assure a better quality.

  • Low Priority
    • There are no Low Priorities for Testing.
